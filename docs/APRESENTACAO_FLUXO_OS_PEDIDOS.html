Guia visual para cliente e admin

Sistema de OS, pedidos, financeiro, status e rastreio

Esta apresentacao explica o que acontece quando um pedido de medicamento e pago, onde a ordem de servico nasce, como a logistica e notificada e como o admin acompanha dinheiro, status e entrega.

Visao geral Fluxo ponta a ponta OS de logistica Admin Cliente Como validar Docs

O pedido nao fica solto no banco

Depois do pagamento aprovado, o sistema registra o pedido, gera uma OS de logistica e abre o acompanhamento para admin e cliente.

1
Pedido

Venda registrada

O checkout salva o pedido em orders com itens, endereco, frete, total, pagamento e status inicial.

OS

Logistica acionada

Cada pedido salvo chama sendLogisticsServiceOrder, criando uma OS em logistics_service_orders e notificando a logistica.

Acompanhamento

Admin e cliente acompanham

O admin gerencia status e rastreio. O cliente ve o progresso em tempo real na tela de detalhes do pedido.

Fluxo ponta a ponta

A ordem de servico nasce automaticamente apos a confirmacao do pagamento. O admin nao precisa consultar o banco para descobrir pedido novo.

2

Checkout

Cliente confirma endereco, frete e pagamento.

Pagamento aprovado

Cielo ou PIX mock retornam sucesso para o checkout.

Pedido salvo

O sistema grava um ou mais pedidos em orders.

OS criada

Cada pedido chama a criacao de OS da logistica.

Admin opera

Equipe separa, atualiza status e informa rastreio.

Cliente acompanha

Status e rastreio aparecem na area do pedido.

O que entra na OS

A OS e pensada para a equipe de logistica separar, embalar e enviar sem abrir telas tecnicas.

3
Dados enviados

Informacoes da ordem

PedidoNumero do pedido gerado no checkout.
ClienteNome, email, telefone quando disponivel e endereco de entrega.
ItensMedicamentos e quantidades daquele pedido.
PrioridadeNormal, com observacao de medicamento controlado quando aplicavel.
Garantia

Mesmo se o email falhar, a OS fica registrada

O backend tenta criar e notificar. Se a notificacao externa falhar, o fallback registra a OS em logistics_service_orders. Assim a operacao ainda consegue auditar o pedido.

Ajuste feito: quando o checkout divide itens por receita, cada pedido salvo agora gera sua propria OS, com os itens corretos daquele pedido.

Como o admin usa

O painel administrativo centraliza financeiro, pedidos, status e rastreio.

4

Financeiro

Acesse /admin/financeiro.
Veja MRR, receita avulsa, receita consolidada, ticket medio e CSV.
Pedidos pagos em processing, shipped e delivered entram na receita avulsa.

Pedidos

Acesse /admin/pedidos.
Veja cliente, itens, valor, endereco e status.
Atualize status conforme enum: pending, processing, shipped, delivered, cancelled.

Rastreio

Informe o codigo de rastreio no pedido.
O sistema normaliza e salva transportadora/eventos.
O cliente recebe status atualizado e pode abrir o link externo dos Correios.

Como o cliente entende

A tela do pedido transforma a operacao interna em uma linha de acompanhamento simples.

5
Status

Linha do pedido

O cliente ve o pedido como pago, em separacao, enviado ou entregue. O realtime conectado atualiza a tela quando o status muda.

Rastreio

Codigo e eventos

Quando o admin informa rastreio, a tela mostra codigo, ultimo evento, data de consulta e botao para acompanhar externamente.
