Cielo · PIX · Produção

Configurar PIX em produção

Passo a passo para colocar o pagamento PIX da Cielo no ar com pedidos reais. Foco em produção — para dev/sandbox veja o guia completo em CIELO_PIX_CONFIGURACAO.md.

Pronto para produção

Como o pagamento PIX flui no sistema

1 Cliente Escolhe PIX no checkout
2 Servidor POST para Cielo gera QR Code
3 Front Exibe QR Code e faz polling 5s
4 Banco do cliente Cliente paga pelo app
5 Cielo Polling ou webhook confirma
6 Supabase Pedido marcado como pago

⚠ Pré-requisitos antes de começar

  • Conta Cielo E-commerce de produção — contratação em cielo.com.br/e-commerce. Pode levar dias úteis de aprovação comercial.
  • MerchantId e MerchantKey de produção — Cielo envia por e-mail após aprovação.
  • Chave PIX da empresa (CNPJ ou aleatória) para vincular à loja Cielo. Sem isso, a Cielo rejeita o pagamento.
  • Backend já publicado no Render com URL pública estável (ex: https://novita-telemedicina-3uv8.onrender.com).
1
Crítico

Habilitar PIX no portal Cielo

PIX não vem habilitado por padrão. Sem este passo, o request retorna erro de validação mesmo com as credenciais corretas.

  1. Acesse minhaconta2.cielo.com.br com sua conta de produção
  2. Menu Loja → Meios de pagamento (o nome pode variar)
  3. Localize Pix e clique em Habilitar
  4. Vincule sua chave PIX (CNPJ ou aleatória da conta bancária da empresa)
  5. Aguarde validação automática com seu banco — pode levar de algumas horas a 1 dia útil
  6. Confirme a aprovação por e-mail antes de seguir
⚠ Bloqueante: sem chave PIX vinculada e validada, a Cielo retorna ProviderError em produção.
2
Configuração

Configurar variáveis de ambiente no Render

Dashboard Render → seu serviço backend → Environment. Adicione/atualize:

VariávelValor
CIELO_MERCHANT_IDUUID de produção recebido por e-mail
CIELO_MERCHANT_KEYChave de API de produção
CIELO_SANDBOXfalse
CIELO_SIMULATEfalse (ou ausente)
CIELO_WEBHOOK_TOKENToken aleatório — gere com openssl rand -hex 32
⚠ Atenção: CIELO_SIMULATE=true em produção gera QR Code falso. Mantenha em false ou remova a variável.

O Render reinicia o serviço automaticamente após salvar.

3
Validação

Validar que as credenciais foram carregadas

Após o restart, confira o log de boot do servidor no Render. A linha de status deve aparecer:

║  Pagamentos: PRODUCTION | Cielo <abc12345>...     ║
║  E-mails:    RESEND (live)                   ║
║  Rastreio:   painel/env                      ║
║  Supabase:   conectado                       ║

Se aparece PRODUCTION e os primeiros caracteres do MerchantId, está OK. Se aparece não configurado, o env não está sendo lido — revise o painel do Render.

4
Recomendado

Configurar webhook da Cielo

Sem webhook, se o cliente fechar o navegador antes do polling de 5s detectar o pagamento, o pedido fica "aguardando pagamento" eternamente. O webhook garante reconciliação assíncrona.

  1. Portal Cielo → Loja → Notificações (ou Webhooks)
  2. Adicione esta URL de notificação: 
    https://<seu-host-render>/api/cielo/webhook?token=<CIELO_WEBHOOK_TOKEN>
  3. Use exatamente o mesmo token que você setou no env do Render
  4. Selecione os eventos:
    • Mudança de status do pagamento (obrigatório)
    • Pagamentos recorrentes (se houver assinaturas)
  5. Salve. A Cielo envia um POST de teste — confira o log: 
    [Cielo Webhook] <paymentId> → status 12 (Aguardando)
Se aparecer 401 Token inválido, o token na URL não bate com o env do Render.
5
Frontend

Apontar o front para o backend de produção

O frontend precisa saber a URL pública do backend Render. Configurado em .env.production:

VITE_LOCAL_SERVER_URL=https://<seu-host-render>.onrender.com

O valor atual é https://novita-telemedicina-3uv8.onrender.com. Se o serviço Render tiver outra URL, atualize e refaça o build.

Também é preciso CORS no backend liberando o domínio do front. Setar no Render:

CORS_ORIGINS=https://novita.migrai.com.br,https://novitahomecare.com.br

Sem isso, o navegador bloqueia as chamadas com erro CORS no console.

6
Crítico

CPF do pagador é obrigatório no PIX

Por exigência do BACEN, todo pagamento PIX precisa do CPF do pagador. A Cielo rejeita transações sem CPF.

No projeto:

  • O signup já exige CPF com validação de regex (src/pages/Auth.tsx) — usuários novos sempre têm.
  • O checkout de medicamentos tem um gate: usuários antigos sem CPF cadastrado são redirecionados para /perfil para completar o cadastro antes de pagar.
  • O CPF é enviado para a Cielo como Customer.Identity + IdentityType="CPF".
✓ Implementado: esse gate já está no código. Nenhuma ação manual necessária — só validar com um usuário antigo (criado antes da regra) que o fluxo redireciona corretamente.
7
Validação

Smoke test em produção

Faça um pagamento real de R$ 1,00:

  1. Como cliente, adicione um medicamento ao carrinho
  2. Checkout → escolha PIX → QR Code aparece (vindo da Cielo, base64)
  3. Pague pelo app do seu banco
  4. Em até 5s (polling) ou imediato (webhook), o pedido vira payment_status='paid'

Confira os 4 lugares:

OndeO que ver
Portal CieloTransação aprovada
Extrato bancárioValor creditado (geralmente D+1)
Supabaseselect id, payment_status, payment_id from orders order by created_at desc limit 1;
Log do servidor[Cielo] ✓ Pedido reconciliado: payment_id=... → paid

✓ Checklist de Go-Live

  • Conta Cielo de produção contratada e aprovada
  • PIX habilitado no portal Cielo
  • Chave PIX vinculada e validada pelo banco
  • Variáveis configuradas no Render: CIELO_MERCHANT_ID, CIELO_MERCHANT_KEY, CIELO_SANDBOX=false, CIELO_WEBHOOK_TOKEN
  • Variável CORS_ORIGINS no Render liberando o domínio do front
  • Servidor reiniciado, log mostra PRODUCTION | Cielo <id>...
  • Webhook configurado no portal Cielo com o token correto
  • Frontend buildado com VITE_LOCAL_SERVER_URL apontando para o backend Render
  • Gate de CPF no checkout testado: usuário sem CPF é redirecionado para /perfil
  • Smoke test de R$ 1,00 passou (pedido vira "paid" automaticamente)

Erros comuns e como resolver

SintomaCausaSolução
"Cielo credentials not configured" no checkout Env não está setado no Render Verificar dashboard Render → Environment, redeploy se preciso
Cielo retorna 401 / 403 MerchantId/Key são sandbox, não produção Trocar pelas credenciais que vieram no e-mail de produção
Erro de validação no POST de pagamento PIX não habilitado na loja Voltar ao Passo 1 — habilitar no portal Cielo
Cielo retorna ProviderError Chave PIX não vinculada ou ainda não validada pelo banco Aguardar validação ou reenviar no portal
Cliente paga, pedido não vira "paid" Webhook não configurado e cliente fechou aba antes do polling Configurar webhook (Passo 4)
Webhook recebe 401 nos logs Token na URL ≠ CIELO_WEBHOOK_TOKEN Conferir token nos dois lados (Render e portal Cielo)
QR Code é um placeholder estranho CIELO_SIMULATE=true em produção Setar CIELO_SIMULATE=false ou remover variável
Erro CORS no console do navegador CORS_ORIGINS não inclui o domínio do front Setar CORS_ORIGINS=https://novita.migrai.com.br no Render
"Servidor de pagamento não configurado" VITE_LOCAL_SERVER_URL vazio no build Setar no .env.production e refazer build/deploy do front
Cielo rejeita PIX com erro de Identity Usuário sem CPF cadastrado Gate de CPF já redireciona para /perfil — confirmar que está funcionando
